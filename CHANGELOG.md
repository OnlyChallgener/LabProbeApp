# LabProbe 更新日志

## v0.10.9 build139

- 路由器“实时状态”新增“存储”卡片，读取快速信息中的磁盘利用率；五张实时卡片保持单行并支持横向滑动。
- 设备主机名、名称或备注中包含 `IPC` 时优先识别为“摄像头”，避免 `IPC` 中的 `PC` 被误判为台式电脑。
- 设备主机名、名称或备注中包含 `KIDSWATCH` 时识别为“儿童手表”。
- 独立的 `PC` 仍按原有逻辑识别为“台式电脑”；其他设备识别与页面逻辑不变。

## v0.10.8 build138

- 设置页的 HTTP Hub 地址改为简洁保存与显示，例如 `192.168.5.46:58443`；实际请求时自动补全 `http://`，HTTPS 地址保持原样。
- WAN 详情调整为“历史流量 / DNS”同排、“网关 / MTU”同排，并移除历史流量小字。
- 路由器上下行速率进一步缩小并强制在同一行显示 Kbps、Mbps 或 Gbps。
- 评分细则页标题字号下调，重做评分节点、得分卡和 Agent 卡图标。
- Rust Agent 卡新增“一键清理”，可清理 Agent 全部备份、非必要日志及失效临时安装文件，并返回清理内容和回收空间。
- 修复 Hub 重启后凭证刷新序号可能与 LabRelay 已确认序号重复，导致宽带账号和密码一直显示 `--`。
- 网络配置展开区第一张小卡由“LAN IP”改为“备注”，读取 `dev_config network` 的 `wan[].service`。
- 路由器状态卡片重新设计光圈、弧线和网络节点，编辑图标紧邻设备名称。
- 上传/下载速率缩小并显示 Kbps、Mbps 或 Gbps；新增 IPv4 / IPv6 连接数。
- 实时状态数值进一步缩小。
- WAN 信息将子网掩码替换为历史上传/下载流量，宽带接口替换为 DNS。
- AP 信息将管理 IP 替换为工作模式，将软件版本替换为信道利用率（如 56% / 8%）。
- 网络配置顶部摘要保持 LAN IP / 子网掩码 / VLAN ID 不变。


## v0.10.7 build137
- 首页网络健康卡中的“出口”入口改为进入独立路由器状态页面。
- 新增路由器状态仪表盘：温度、CPU、内存、运行时间、在线终端、WAN 速率、网络配置及物理端口状态。
- 路由器状态使用 MQTT 实时更新，断线时由 HTTP 自动兜底，并支持手动刷新。
- 端口区域只保留端口图标和名称，不显示重复的速率、双工模式及“已连接”文字。
- 本次不调整其他页面 UI 和原有网络工具逻辑。

## v0.10.5 build133
- MQTT 前台实时同步，断线自动重连并由 HTTP 增量同步兜底，每 5 分钟完整校准。
- APP 设置仅保留 APP_TOKEN；HOOK_TOKEN 仅供 LabRelay、Lucky 与 Webhook 使用。
- 首页“关注终端”与设备页使用同一关注筛选，取消关注后立即消失。
- 首页健康分和评分细则路由器的动态光晕向周围扩散更明显。
- Rust Agent 上一次检查结果持久保存，退出评分页面后不再丢失。
- APP 版本显示和更新判断统一读取 Gradle BuildConfig，避免多处版本号不一致。
- 清理重复 README 和旧版独立 RELEASE 文档。

## v0.10.0-alpha1 build126
- 全局 UI V2：新的淡蓝背景、雾面卡片、底部导航、统一弹层和输入组件。
- 工具页功能图标改为全新的非 3D 图标体系；设备图标素材、设备识别和设备逻辑完全保留。
- 端口映射编辑页重新排版：短字段并排、长 IPv6 全宽、文字垂直居中、窄屏自动改单列。
- 6→6 设备选择器按模式显示设备 IPv6 与后 64 位；无可用 IPv6 的设备会明确提示并隐藏。
- 定时映射新增可重复有效期字段，规则到期后再次启动按上一次时长重新计时（需 Hub v0.8.3）。
- SSH 保持现有“连接与命令 + 淡色结果卡片”设计，不改成整页黑色终端。
- Ping、漫游、流量等图表采样、坐标、事件和曲线逻辑未修改。
