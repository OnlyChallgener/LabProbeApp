# LabProbe 更新日志

## v0.10.15 build146（当前分支）

- 路由器状态页接入 Hub 已发布的 MQTT `dashboardTopic`，Hub 收到路由器 WSS 快照后可立即推送到 APP，不再等待 15 秒轮询。
- APP 记录最近一次仪表盘 MQTT 到达时间；2.5 秒内有推送时以 MQTT 为主，推送中断或 MQTT 不可用时自动改为每 1 秒读取 Hub 内存快照。
- 修复生成脚本仅凭更新日志文字判断补丁已应用，导致实际 APK 仍保留 15/20 秒刷新周期的问题；实时补丁现在固定最后执行并带构建断言。
- 手动刷新只等待仪表盘刷新，宽带账号密码改为后台刷新，不再让按钮额外阻塞约 5 秒。
- 保留 build145 的 NAT 双协议历史、圆角下拉框、稳定按钮状态和网络自检历史结果修复。
- 配套 Hub 仍为 0.9.15，LabRelay 仍为 0.2.10。

## v0.10.15 build145

- 路由器状态页改为约每 1 秒读取 Hub 内存快照，WSS 快数据不再被终端列表 RPC 阻塞。
- APP 保存上一份可用路由器仪表盘，并对新快照做深度合并；刷新、短暂断线或空响应时不再把页面内容清空成白屏。
- 手动刷新立即返回，宽带账号密码刷新转入后台，不再让刷新按钮长时间停留在“刷新中”。
- Hub 路由连接、同步和异常提示统一改为中文，区分“数据正常”“正在同步”“连接恢复中”。
- 路由 NAT 最近检测按协议保存：RFC3489 与 RFC5780 各保留最新一条；只有一种协议记录时仅显示一条。STUN 端口不参与历史分组。
- NAT 的 STUN、检测协议与出口接口弹层统一为圆角白色；NAT、Beta 与网络自检按钮补全禁用态颜色和透明加载轨道，修复灰框与白色方块。
- 网络自检结果持久保存在 APP 本地，重新进入页面先显示上次有效结果；Hub 瞬时返回空结果时不再覆盖历史结果。
- 首页 SSH 快捷小卡保持浅灰色。
- 配套 Hub 版本为 0.9.15；LabRelay 0.2.10 改为低流量直连 Hub 模式，需要同步更新。

## v0.10.13 build143

- 首页保留独立“路由设置”总入口；健康卡小入口由 VPN 改为 SSH，半宽 VPN/STUN 卡改为 DDNS 并显示记录、启用和异常数量。
- 路由器设置新增原生 NAT 诊断，支持 RFC 3489、RFC 5780、WAN/WAN1、STUN 服务器选择、检测日志和最近 10 次历史。
- 路由器设置新增 ReyeeOS Beta 在线版本检查，显示当前版本和可用版本；实际安装负载确认前不执行升级。
- 路由器状态页改为约每 2 秒读取 Hub 内存中的 WebSocket 快照，提升实时速率、连接数、CPU 和内存更新速度。
- WOL 设备与每日总结标题略微缩小；每日总结终端记录之间增加少量间距。
- 完整宽带账号密码继续通过现有 LabRelay 按需读取，不写入普通状态缓存；LabRelay 版本保持 0.2.9。

## v0.10.12 build142

- 路由器状态接入 Hub 的 Reyee eWeb `/ws` WebSocket 数据，显示 CPU、内存、温度、运行时间、WAN 速率、连接数、端口与无线状态。
- 首页新增独立“路由设置”卡片，路由器配置功能集中到二级页面，路由器状态仍保持独立入口。
- 修复 DDNS 卡片和更多菜单点击闪退，异常状态不再整卡标红。

- APP 与 LabRelay 版本号保持不变；本次主要更新 Hub 登录实现。

## v0.10.10 build140

- SSH 执行结果弹窗中，命令文本支持按需选择并复制部分内容。
- SSH 当前页面仍仅展示最近 6 条结果，完整执行记录独立保存最多 30 条。
- 新增 SSH 执行记录三级页面，显示条数、占用空间并支持左滑删除。
- SSH 页面与“连接与命令”标题字号略微缩小。
- 路由器存储利用率改为读取 Relay 每小时采集一次的 `df -h /overlay` 结果。
- 实时状态卡片主数值字号再次小幅缩小。

## v0.10.9 build139

- 路由器“实时状态”新增“存储”卡片，读取快速信息中的磁盘利用率；五张实时卡片保持单行并支持横向滑动。
- 设备主机名、名称或备注中包含 `IPC` 时优先识别为“摄像头”，避免 `IPC` 中的 `PC` 被误判为台式电脑。
- 设备主机名、名称或备注中包含 `KIDSWATCH` 时识别为“儿童手表”。
- 独立的 `PC` 仍按原有逻辑识别为“台式电脑”；其他设备识别与页面逻辑不变。

## v0.10.8 build138

- 设置页的 HTTP Hub 地址改为简洁保存与显示，例如 `192.168.5.46:58443`；实际请求时自动补全 `http://`，HTTPS 地址保持原样。
- WAN 详情调整为“历史流量 / DNS”同排、“网关 / MTU”同排，并移除历史流量小字。
- 路由器上下行速率进一步缩小并强制在同一行显示 Kbps、Mbps 或 Gbps。
- 评分细则页标题字号下调，重做评分节点、得分卡和 Agent 卡图标。
- Rust Agent 卡新增“一键清理”，可清理 Agent 全部备份、非必要日志及失效临时安装文件，并返回清理内容和回收空间。
- 修复 Hub 重启后凭证刷新序号可能与 LabRelay 已确认序号重复，导致宽带账号和密码一直显示 `--`。
- 网络配置展开区第一张小卡由“LAN IP”改为“备注”，读取 `dev_config network` 的 `wan[].service`。
- 路由器状态卡片重新设计光圈、弧线和网络节点，编辑图标紧邻设备名称。
- 上传/下载速率缩小并显示 Kbps、Mbps 或 Gbps；新增 IPv4 / IPv6 连接数。
- 实时状态数值进一步缩小。
- WAN 信息将子网掩码替换为历史上传/下载流量，宽带接口替换为 DNS。
- AP 信息将管理 IP 替换为工作模式，将软件版本替换为信道利用率（如 56% / 8%）。
- 网络配置顶部摘要保持 LAN IP / 子网掩码 / VLAN ID 不变。
