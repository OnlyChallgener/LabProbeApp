# LabProbe 更新日志

## v0.10.15 build151（当前分支）

- 配套 Hub 0.9.18 与 LabRelay 0.2.11，真实数据改为路由器本地 `dev_sta` 采集后推送到 Hub 内存。
- APP 保留每秒读取两个轻量接口，但不再让新样本瞬间跳变；网速、CPU、内存和温度在相邻真实样本之间做 900ms 平滑过渡。
- 终端列表只按 MAC 平滑上传和下载速率；连接数、上线和离线状态仍立即使用真实整数与真实事件。
- 平滑层每 200ms 渲染一次，只在显示值真正变化时更新 Compose 状态，避免整页无意义重组。
- 同一个 `sampleEpochMs` 不会重复启动动画，旧样本也不能覆盖新样本。
- 样本年龄超过 6 秒后停止插值并保持最后真实目标；超过 8 秒显示过期状态，不预测、不随机生成网速。

## v0.10.15 build150

- APP 改用 `/api/router/realtime` 与 `/api/devices/realtime` 两个轻量接口，路由和终端实时请求互不等待。
- 终端实时结果只按 MAC 覆盖上传、下载和连接数，不重新解析完整设备资料。
- MQTT 只保留 revision 与在线状态，不再传输完整 Dashboard 大 JSON。
- 完整 Dashboard 仅用于静态资料、配置校准与手动刷新。

## v0.10.15 build147

- 修复“关于”卡片的版本日志仍显示 v0.10.6，而 APK 版本已经升级的问题。
- “关于”卡片的发布标题改为直接读取 `BuildConfig.VERSION_NAME` 与 `BuildConfig.VERSION_CODE`，以后构建号变化时自动同步。
- 版本日志修复脚本固定在所有旧生成脚本之后最后执行，旧日志无法再次覆盖当前版本。
- CI 强制检查 APK 版本、关于页版本标题与旧版日志残留；任一不一致时停止生成安装包。

## v0.10.15 build146

- 路由器状态页接入 Hub 已发布的 MQTT `dashboardTopic`，Hub 收到路由器 WSS 快照后可立即推送到 APP，不再等待 15 秒轮询。
- APP 记录最近一次仪表盘 MQTT 到达时间；2.5 秒内有推送时以 MQTT 为主，推送中断或 MQTT 不可用时自动改为每 1 秒读取 Hub 内存快照。
- 修复生成脚本仅凭更新日志文字判断补丁已应用，导致实际 APK 仍保留 15/20 秒刷新周期的问题。
- 手动刷新只等待仪表盘刷新，宽带账号密码改为后台刷新。

## v0.10.15 build145

- 路由器状态页保存上一份可用路由器仪表盘，并对新快照做深度合并；短暂断线或空响应时不再清空页面。
- Hub 路由连接、同步和异常提示统一改为中文。
- 路由 NAT 最近检测按协议保存：RFC3489 与 RFC5780 各保留最新一条；STUN 端口不参与历史分组。
- NAT 的 STUN、检测协议与出口接口弹层统一为圆角白色；修复按钮灰框和加载白块。
- 网络自检结果持久保存在 APP 本地，重新进入页面先显示上次有效结果。

## v0.10.13 build143

- 首页保留独立“路由设置”总入口；健康卡小入口由 VPN 改为 SSH，半宽 VPN/STUN 卡改为 DDNS。
- 路由器设置新增原生 NAT 诊断，支持 RFC 3489、RFC 5780、WAN/WAN1、STUN 服务器选择和检测日志。
- 路由器设置新增 ReyeeOS Beta 在线版本检查。
- WOL 设备与每日总结标题略微缩小；每日总结终端记录之间增加少量间距。
