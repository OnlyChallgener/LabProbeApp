# LabProbe 更新日志

## v0.10.15 build152（当前分支）

- 轻量路由接口成功响应作为 APP 与 Hub 的实时连接租约，租约有效期 5 秒。
- 租约失效后立即停止终端实时请求、200ms 平滑渲染和缓存计算。
- 断线期间只保留一个路由探测，并按 3 秒、5 秒、10 秒、15 秒逐级退避。
- 路由探测恢复后自动重新开启每秒真实样本、终端实时请求和平滑显示。
- APP 退到后台时继续沿用 `stopRealtime()`，立即取消全部实时任务并清空连接租约。
- Hub 端 5 秒需求租约到期后，LabRelay 0.2.11 也会停止本地高频采集和推送，恢复长轮询。

## v0.10.15 build151

- 配套 Hub 0.9.18 与 LabRelay 0.2.11，真实数据改为路由器本地 `dev_sta` 采集后推送到 Hub 内存。
- APP 保留每秒读取两个轻量接口，但不再让新样本瞬间跳变；网速、CPU、内存和温度在相邻真实样本之间做 900ms 平滑过渡。
- 终端列表只按 MAC 平滑上传和下载速率；连接数、上线和离线状态仍立即使用真实整数与真实事件。
- 平滑层每 200ms 渲染一次，只在显示值真正变化时更新 Compose 状态。
- 同一个 `sampleEpochMs` 不会重复启动动画，旧样本也不能覆盖新样本。
- 样本年龄超过 6 秒后停止插值并保持最后真实目标；超过 8 秒显示过期状态，不预测、不随机生成网速。

## v0.10.15 build150

- APP 改用 `/api/router/realtime` 与 `/api/devices/realtime` 两个轻量接口，路由和终端实时请求互不等待。
- 终端实时结果只按 MAC 覆盖上传、下载和连接数，不重新解析完整设备资料。
- MQTT 只保留 revision 与在线状态，不再传输完整 Dashboard 大 JSON。
- 完整 Dashboard 仅用于静态资料、配置校准与手动刷新。
